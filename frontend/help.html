はじめての方向けガイド

このアプリでできること

外国語や日本語の文章を読み込み、キーワード（人名・国名・都市名・組織名）を抜き出したり、 別の言語へ翻訳したりできます。インターネット不要・CPU だけで動きます（モデルは事前配置）。

操作の流れ（おすすめ動線）

上から順に進めると迷いにくいです。

  1. 1

    本文を入れる

    左パネル「本文 / Source」にテキストを用意します。

    • PDF / テキストを添付 … ツールバーの添付ボタンからファイルを選ぶ
    • 貼り付け … メモ帳やブラウザからコピーし、本文欄にペースト
    • 手入力 … そのまま入力しても OK

    PDF はテキスト抽出できるもの向けです。スキャン画像だけの PDF は文字が取れないことがあります。

  2. ↓
  3. 2

    言語を確認する

    翻訳パネルで元言語 → 訳出言語を選びます。元言語は自動判定も使えます。

  4. ↓
  5. 3

    やりたい処理を選ぶ

    キーワード抽出

    「キーワード抽出」を押すと、本文から人名・国・都市・組織を一覧に出します。

    翻訳

    「翻訳」を押すと、文ごとに翻訳が進み、進捗バーで状況が分かります。

    初回やモデル切替時は、読み込み画面が出ることがあります。完了まで待ってください。

  6. ↓
  7. 4

    結果を確認・コピーする

    • 翻訳後、本文と訳文の対応ブロックをクリックすると、同じ箇所がハイライトされます
    • 各パネル下のコピーで、本文／訳文をクリップボードへコピーできます
    • キーワード一覧から、必要な語を確認・再利用できます
PDF / テキスト貼付 → 本文パネル → 言語選択 → 抽出 or 翻訳 → 確認・コピー

画面の見かた（ざっくり）

  • 左 … 元の文章（Source）
  • 右上 … 翻訳の言語・実行、翻訳結果
  • 右下付近 … キーワード抽出の結果
  • 最上部右 … テーマ（Light / Dark）と、このヘルプ（本マーク）

使っているモデルの解説

キーワード抽出（NER）

名前GLiNER multilingual medium（ONNX int8）
役割文章中の固有表現をラベル付きで検出
抽出ラベルPerson（人名）、Country（国）、City（都市）、Organization（組織）
実行方式ONNX Runtime・CPU のみ・量子化（int8）で軽量化
RAM 目安約 0.5〜1.0 GB

「ラベルを指定して探す」タイプの NER です。ニュース記事・レポートなど、固有名詞が多い文章向きです。 誤検出・取りこぼしはあり得ます。重要な案件では目視確認を推奨します。

翻訳（MT）

名前NLLB-200 distilled（CTranslate2 int8）
役割多言語間の機械翻訳（200 言語系ファミリー）
切替UI で 600M / 1.3B を選択
実行方式CTranslate2・CPU のみ・int8。文単位に分割して逐次翻訳
ライセンスNLLB は CC-BY-NC（非商用）。社内・個人利用前提。商用転用は不可
バリアント 向き RAM 目安 速度・品質の目安
600M（既定） 日常利用・メモリ節約 約 1.0〜1.5 GB より軽い。短い記事なら十分なことが多い
1.3B 品質優先・メモリ余裕あり 約 2.0〜2.5 GB やや重いが、用語や文脈の安定感が出やすい傾向

CPU では 1 文あたり数秒かかることがあります。長い文書ほど時間が増えます。 専門用語・略語・固有名は、用語集・後処理で補正している場合がありますが、機械翻訳の限界は残ります。

メモリと排他ロード

NER と MT は同時に常駐しません（どちらかを使うときに切り替えてロード）。 そのため、ピーク RAM を抑えやすくなっています。切替のたびに読み込み時間が発生します。

うまくいかないとき

  • モデル未配置 … 起動時や実行時にエラーが出ます。配布物の models/（または model/）を確認してください
  • 翻訳が遅い … 正常です。進捗バーを見て待つか、600M を試してください
  • PDF が空 … 画像 PDF の可能性。テキスト PDF にするか、手動で貼り付けてください
